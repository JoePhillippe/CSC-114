# Projects vs. Platform — A Builder's Comparison

**Same bot, two platforms. Here's what I noticed.**

## What was easier on claude.ai (Session 0)?
Claude.ai was easier to setup and create agent.  It would also create and edit files I plan to post on github.

## What was easier on platform.claude.com (Session 1)?
Platform was hard to setup environment but I had much more control after setup. I needed help from Claude.ai to create python code that I ran in Spyder IDE to upload a file. This is expected since developer environments are always improving so the GUI is changing. 

## If I had to deploy CSC-114-InfoBot to 500 students next semester,
which platform would I use, and why?
     I would use Claude Platform for something quick like CSC=114 InfoBot to 500 students especially when working with a team that has members that cannot code. Claude Platform combines developing the AI Agent and hosting but does intergrate with github. Claude.ai was better for quick prompt testing and providing support to work with Claude Platform. 
     I have used Claude.ai for planning steps and prompt creatation for Claude Code on VS Studio Code that created a github clone folder then pulled and push then commited files to develop an AI Agent.  I hosted the AI agent on a flask webpage. I used Render web host to provide a public environment which intergrated with github. Using github would be better for a team project.
     I attended a conference for developing AI Agents for Community College Instructors and none of the presenters or attendees could code but had developed many agents using Gemini Gems.
     But I want to see the python, HTML and json script development for larger projects. I debugged a lot of design and coding errors myself as I worked with Claude Code in my lasted larger AI Agent.  All three working enviroments are usefull depending on the type on AI Agent being create and the phase of the project.
